# Part 4: 문서 처리 라이브러리 완전 가이드

> **이 파트는 이 가이드북의 심장**입니다. 지금 당신이 읽고 있는 이 페이지도, 그리고 이 가이드북의 Word/PDF 버전도 **전부 이 파트에서 소개하는 라이브러리로 자동 생성**되었어요.

---

## 🪞 메타 컨셉: 가이드북이 스스로를 만들다

이 프로젝트의 가장 재미있는 부분은 **자기 지시성**(self-reference)입니다.

```
src/*.md  →  python-docx / WeasyPrint / 마크다운 파서  →  Guidebook.docx / Guidebook.pdf
   ↑                                                              ↓
   └──────────────── 이 가이드북이 소개하는 라이브러리들 ─────────┘
```

즉, **가이드북이 가이드북 자신을 생성**합니다. "문서 자동화를 배우려면 이 라이브러리를 써라"라고 말하는 가이드북이, 바로 그 라이브러리들로 만들어지는 거예요. 셀프 호스팅 컴파일러와 같은 느낌이죠.

!!! tip "💡 직접 체험해보세요"
    이 섹션을 다 읽고 나면 [4-4. 문서 자동화 파이프라인](04-pipeline.md)에서 `build_guidebook.py`를 직접 실행해서 **당신 이름이 표지에 박힌 PDF**를 만들어보는 미니 튜토리얼이 기다리고 있어요.

---

## 왜 문서 처리 라이브러리를 배워야 하는가

대부분의 실무는 **문서로 시작해 문서로 끝납니다**.

- 주간 보고서를 매번 손으로 Word에 타이핑하시나요?
- Excel 양식 100개를 하나씩 채우고 계신가요?
- PDF에서 표 데이터를 복붙하고 계신가요?
- PowerPoint 템플릿에 매주 같은 차트를 그리고 계신가요?

이 모든 반복 작업을 **30줄의 Python 코드**로 없앨 수 있습니다. 그게 이 파트의 약속이에요.

### 실제 활용 사례

| 상황 | 기존 방법 | 라이브러리 활용 후 |
|---|---|---|
| 주간 매출 보고서 50개 생성 | 2시간 수동 작업 | 3초 자동 생성 |
| 설문 PDF에서 응답 추출 | 일일이 복붙 | `fitz.get_text()` 한 줄 |
| 엑셀 100개 시트 취합 | 반나절 | `openpyxl` 루프 한 번 |
| 발표자료 20장 양산 | 하루 | `python-pptx` 템플릿 |

### 어느 언어부터 시작할까요?

!!! tip "💡 언어 선택 가이드"
    - **Python 추천** (대부분의 독자): 가장 직관적이고, 라이브러리 생태계가 압도적입니다.
    - **Java 선택**: 백엔드가 Spring Boot이고 서버에서 문서를 생성해야 한다면.
    - **JavaScript 선택**: 브라우저에서 클라이언트 사이드로 문서를 만들거나, Node.js 서버를 쓴다면.

이 가이드북은 Python 3.11 + `python-docx` + `WeasyPrint`로 만들어졌습니다.
