    Part 3: 프로젝트 및 공모전 활용¶

    "이론은 충분히 배웠는데, 이제 뭘 만들죠?" 라는 질문에 답하는 파트예요. 여기서는 실전 프로젝트 경험을 쌓고, 포트폴리오를 만들고, 오픈소스에 기여하고, 해커톤에서 우승하는 데 필요한 GitHub 레포들을 소개합니다.

    이 파트에서 배우는 것¶

    • 풀스택 프로젝트 템플릿을 활용해 1주일 안에 데모 가능한 앱 만들기
    • 오픈소스에 첫 PR을 날리는 전 과정 (Fork → Branch → Commit → PR → Merge)
    • 해커톤에서 24시간 안에 MVP를 완성하는 보일러플레이트 활용법
    • Claude Code로 레포를 내 입맛에 맞게 200% 활용하는 프롬프트 패턴

    구성¶

    섹션 주제 난이도
    3-1. 풀스택 프로젝트 템플릿 RealWorld, Full Stack FastAPI Template, Redoc ⭐⭐⭐
    3-2. 포트폴리오 & 오픈소스 기여 First Contributions, Awesome for Beginners, Opensource Guide ⭐⭐
    3-3. 해커톤 & 공모전 Hackathon Starter, AutoGPT, LangChain ⭐⭐⭐⭐

    💡 꿀팁: 이 순서대로 읽으세요

    비전공자/입문자라면 3-2 → 3-1 → 3-3 순서를 추천해요. 오픈소스 첫 PR로 Git 감각을 익힌 뒤, 풀스택 템플릿으로 "완성된 앱"을 경험하고, 마지막에 해커톤용 고급 템플릿으로 넘어가는 흐름이 가장 부드럽습니다.

    ⚠️ 초보자가 가장 많이 하는 실수

    "큰 레포를 처음부터 끝까지 다 이해하려고" 하지 마세요. 10만 Stars짜리 프로젝트는 전문가 수십 명이 몇 년간 만든 것입니다. 핵심 1-2개 파일만 읽고, 나머지는 Claude Code에게 "이 폴더 구조 설명해줘"라고 물어보는 편이 훨씬 빨라요.

    학습 목표 체크리스트¶

    • 풀스택 템플릿 하나를 로컬에서 실행하고 스크린샷 남기기
    • First Contributions 레포에 첫 PR 제출하기
    • 포트폴리오용 README.md 1개를 오픈소스 가이드 스타일로 작성하기
    • 해커톤 스타터 키트로 회원가입/로그인 앱 24시간 내 완성하기
    • LangChain으로 "내 PDF에 질문하기" RAG 데모 만들기
